# SOFTWARE REQUIREMENTS SPECIFICATION (SRS)

## Назва проєкту: FileCraft-обробник файлів

### Інформація про документ:
* **Версія:** v0.1 (draft) 
* **Дата:** 19.03.2026 
* **Команда:** Панасенко, Білоконь, Окол
* **Замовник:** Стартап YoungAnalytics ABB

---

### 1. ВСТУП (INTRODUCTION)
* **Опис:** FileCraft — веб-сервіс для швидкого збереження та обробки файлів за допомогою ШІ. 
* **Замовник:** Стартап YoungAnalytics ABB. 
* **Проблема:** Користувачі витрачають багато часу на пошук основної інформації у файлі.
* **Цільова аудиторія:** Студенти та бізнес-аналітики (вік 17–48, технічно грамотні користувачі). 

### 2. ЗАГАЛЬНИЙ ОПИС (OVERALL DESCRIPTION)
**Актори:** 
* **Гість (незареєстрований):** перегляд файлів до 500 МБ на день без ШІ.
* **Зареєстрований користувач:** обробка файлів до 1 ГБ на день та базові можливості ШІ.
* **Premium:** обробка файлів до 10 ГБ на день та розширені можливості ШІ.
* **Залежності:** SMTP-сервер (email-підтвердження), PostgreSQL (зберігання даних). 

### 3. ФУНКЦІОНАЛЬНІ ВИМОГИ (FUNCTIONAL REQUIREMENTS)
* **FR-01 [Must]:** Система дозволяє реєстрацію через email.
* **FR-02 [Must]:** Зареєстрований користувач може завантажити файл та отримати оброблений звіт.
* **FR-03 [Must]:** ШІ оброблює файли.
* **FR-04 [Should]:** Підтримується пошук файлів за ключовими тегами.
* **FR-05 [Should]:** Файли можна позначати тегами.
* **FR-06 [Could]:** Файл можна зробити публічним та поділитися посиланням.
* **FR-07 [Must]:** Користувач може видалити та коригувати файл.
* **FR-08 [Should]:** Адмін може деактивувати обліковий запис.
* **FR-09 [Must]:** Користувач може купити Premium та використовувати всі його можливості.

### 4. НЕФУНКЦІОНАЛЬНІ ВИМОГИ (NON-FUNCTIONAL REQUIREMENTS)
* **NFR-P-01:** Час відповіді для 95% операцій < 1,5 с при 500 одночасних користувачів.
* **NFR-S-01:** Паролі зберігаються у вигляді argon2-хешів (cost factor ≥ 12).
* **NFR-S-02:** HTTPS обов'язковий (TLS 1.2+).
* **NFR-U-01:** Інтерфейс адаптивний: підтримка екранів 320px–2560px. 
* **NFR-R-01:** Uptime ≥ 99.5% (не більше 22 год/місяць недоступності).
* **NFR-R-02:** Автоматичний backup БД раз на 12 год.

### 5. ОБМЕЖЕННЯ (CONSTRAINTS)
* **C-01:** Back-end реалізується на Node.js (Express.js або NestJS).
* **C-02:** Розгортання — Docker-контейнери на VPS.
* **C-03:** Бюджет: 10 000€ на створення веб-сервісу, до 1000€/рік на інфраструктуру.
* **C-04:** Дедлайн MVP: 3 місяці від дати підписання контракту.
* **C-05:** Відповідність GDPR (права користувача на видалення даних).

### 6. ГЛОСАРІЙ (GLOSSARY)
* **Файл (File):** текстовий документ з заголовком, тілом.
* **Тег (Tag):** коротке ключове слово для знаходження потрібного файлу.
* **MVP (Minimum Viable Product):** мінімальна версія продукту з базовою функціональністю.
* **GDPR:** регуляторний акт ЄС щодо захисту персональних даних.
* **CRUD:** Create, Read, Update, Delete (базові операції над даними).
